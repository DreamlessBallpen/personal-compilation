This comes from the thought of "Can custom made objects in Python be used as dictionary keys?"
The currently collected information is:

(1) Yes, but
(2) You need to set a __hash__ operator, where the attribute passed to the keyword hash is an immutable value from your object.
	-- LEARNING: Only immutable objects can be hashed.
(3) You need to implement the __eq__ operator as well
	-- What you use for the hash keyword would also be used here as one of the criteria that needs to be compared.
	-- I'm not fully sure, but it seems like this is what would be used by the 'in' keyword if you look for it in a list of custom objects of the same class.
